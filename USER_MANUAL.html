Release Analyzer

User Manual & Documentation

1. Getting Started

Welcome to the Release Analyzer. This application helps you track software releases and analyze Service Level Agreement (SLA) compliance based on Jira data.

Launching the App

Open the application from your desktop or Applications folder. Upon launch, you will see the main dashboard where you can create or select a project.

2. Managing Projects

Creating a Project

  1. Click the + New Project button in the sidebar.
  2. Enter a unique project name (e.g., "MyProject 2026").
  3. Press Enter or click "Create".

Importing Data

You can import data in two ways:

  • CSV Upload: Click the "Upload" icon in the top right to import a standard release file.
  • Jira Import: Use the "Import from Jira" button in the sidebar to fetch data directly (requires API Token).

  • Throughput Analysis: Shows the volume of open vs. closed issues over time. Defaults to daily granularity for the current month.
  • Release History: A timeline of releases referencing specific issue keys.

4. Issue Analysis

Switch to the Issue List tab to see granular details.

The Detail Table

The table displays:

  • Key: The Jira Issue Key (e.g., PROJ-123).
  • Priority: The issue priority level.
  • Status: Current status (Done, In Progress, etc.).
  • Reaction Time: Time taken to pick up the issue.
  • Resolution Time: Total time to resolve, excluding "Pause" states.

Click on any row to open a full Detail Modal showing the exact timeline and calculation breakdown.

5. Troubleshooting

Common Issues

  • "Invalid CSV Format": Ensure your Jira export includes the required headers: Issue key, Summary, Created, Resolved, and Priority.
  • "No Data Shown": Check your filters. Ensure the "Month" filter includes the dates of your imported issues.
  • Charts look empty: Try clicking "Reset / Upload New" to clear any stale state and re-upload your file.
